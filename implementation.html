Property requirements

What we need from your property to deploy successfully.

ARIO's implementation programme is designed to minimise operational disruption. The following is required from every property prior to deployment.

  • 01
    A named Revenue Manager
    The designated Revenue Manager will review the Decision Journal daily throughout Shadow mode and attend the Day 14 Shadow Review. In Review mode, the RM holds approval authority for every queued rate push.
  • 02
    PMS access credentials
    API credentials for Opera Cloud, Mews, Cloudbeds, or Apaleo — or access to export structured reservation history for file-transfer integration. ARIO's implementation manager will specify the exact permissions required.
  • 03
    Channel manager credentials
    Credentials for SiteMinder, RateGain, or your native channel manager (Mews Distribution, Opera CRS). All credentials are stored using AES-256-GCM encryption and never transmitted in plaintext.
  • 04
    Rate guardrail parameters
    Floor and ceiling rates per room type, plus any stability thresholds (maximum single-move percentage, minimum nights before rate change). These constrain Autopilot operations for the life of the deployment and can be adjusted at any time.
  • 05
    Comp-set properties
    A list of 3–5 competitor properties for comp-set configuration. ARIO's implementation manager will validate market data availability for each nominated property and advise on alternatives if needed.
Configuration note

Properties without a dedicated Revenue Manager

Review mode can be operated with a General Manager taking the daily approval function. This is a supported configuration — the GM receives a structured morning briefing designed for time-constrained review, with each recommendation accompanied by a plain-language rationale and confidence indicator.

Shadow mode and the Day 14 review are still required under this configuration. ARIO's implementation manager provides additional support throughout the approval orientation period.

Properties operating under GM-approval configuration remain in Review mode until a qualified Revenue Manager is in position. Advancement to Autopilot requires documented sign-off from a credentialed RM.

Operating mode architecture

Three modes. Defined gates. Full control at every stage.

ARIO's operating modes are enterprise controls, not a hierarchy of risk tolerance. Every property begins in Shadow. Progression to each subsequent mode requires documented gate criteria to be met and confirmed.

Mode 01 · Default for all new deployments
Shadow
Observe. Validate. Build confidence.
The full ARIO engine runs against live property data. Every pricing recommendation is generated, scored, and logged in the Decision Journal. Nothing goes live. No rate push is possible in Shadow mode — by design.
  • Full forecasting engine active on live data
  • Every recommendation logged with model attribution and confidence score
  • Plain-language rationale generated for every decision
  • Daily journal review with implementation manager (first 5 days)
  • Formal Shadow Review at Day 14
  • No outbound rate activity possible — hard system constraint
Entry
Phase 1 integration complete, PMS connection confirmed, data validation passed.
Mode 02 · Standard operating mode
Review
Approved decisions. Confirmed execution.
Pricing recommendations are queued for RM approval each morning. The Revenue Manager reviews, approves, or rejects each recommendation before any rate is pushed. Every approved push is confirmed by read-back from the channel manager before being marked complete in the audit trail.
  • Recommendations queued for RM review each morning
  • No rate push without explicit RM approval
  • Channel manager read-back confirmation on every push
  • Full audit trail: recommendation → approval → execution → confirmation
  • Weekly performance reports comparing recommendations against actuals
  • Kill switch active at all times
Gate criteria
14-day Shadow period complete. Shadow Review report signed off. Go/no-go assessment passed by implementation manager.
Mode 03 · Supervised autonomous operation
Autopilot
Within your defined parameters.
ARIO operates autonomously within the guardrails defined at Phase 1. Rate pushes are executed without manual approval, subject to configured floor/ceiling constraints, stability thresholds, and move-size limits. The kill switch remains accessible at all times and stops all outbound rate activity within 45 seconds.
  • Autonomous rate management within configured guardrails
  • Hard floor/ceiling enforcement — no guardrail breach possible
  • Kill switch: all outbound activity stopped within 45 seconds
  • Full Decision Journal maintained in Autopilot — every push logged and explained
  • Monthly performance reviews and model recalibration
  • Multi-property portfolio management and chain rollup available
Gate criteria
30+ days of Review mode data accumulated. Documented RM sign-off confirming eligibility. Dual signoff gate: RM and ARIO implementation manager must both confirm before Autopilot is enabled.
